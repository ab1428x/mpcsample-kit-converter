# mpcsample-kit-converter

Bulk-convert Akai MPC expansion packs (`.xpm` kit definitions) into ready-to-play `.xpj` projects for the **MPC Sample** hardware device.

Each kit is exported as a self-contained `.xpj` project alongside a `KitName_[ProjectData]/` folder containing the kit's WAV files. The `_[ProjectData]` folders are invisible in the MPC device browser, so an entire expansion appears flat — open one folder and see all kit names directly, with no sub-navigation required.

---

## Features

- Reads Akai `.xpm` kit definitions and maps every pad to its correct WAV
- Preserves mute groups from the source XPM
- Auto-detects BPM from kit filenames
- Embeds the kit's demo groove when an `.sxq` sequence file is present alongside the `.xpm` — hit play and the pattern runs immediately at the correct BPM
- Sets correct WAV start/end frame values so every sample plays immediately
- Keeps original WAV filenames — no `A01`/`A02` renaming
- Processes a single kit, one expansion (by path or short name), or all expansions in one command
- Filter or skip kits by name pattern
- Parallel processing with `--workers` for faster bulk generation
- Pure Python stdlib — no dependencies to install

---

## Requirements

- Python 3.10 or later
- A `.xpj` project file exported directly from your MPC device (see [Template](#template) below)

## Understanding the source files

Before running the converter it helps to know what the MPC Content directory looks like and which files do what. All of the source material the converter reads lives in one place on your computer.

### Where the Content directory is

**macOS**
```
/Library/Application Support/Akai/MPC/Content/
```
This is the system-level Library folder (`/Library`), not your user Library (`~/Library`). If you open Finder, press **Cmd+Shift+G**, and paste that path, you'll land right in it.

**Windows**
```
C:\ProgramData\Akai\MPC\Content\
```
`ProgramData` is a hidden folder by default. To reach it, open File Explorer, click the address bar, type that path, and press Enter. You can also press **Win+R**, type `%ProgramData%\Akai\MPC\Content`, and hit Enter.

Both locations are set automatically — you only need `--content-dir` if you've moved your content to a custom drive.

---

### What's inside

Each installed expansion pack gets its own folder inside Content, named with a reverse-domain identifier:

```
Content/
├── com.akaipro.mpc.expansion.golddust/
├── com.akaipro.mpc.expansion.nuskillz/
├── com.akaipro.mpc.expansion.soulprovider2/
└── ...
```

Inside each expansion folder you'll find a flat mix of files:

```
com.akaipro.mpc.expansion.golddust/
│
├── HipHop-Kit-GD Rawcutz Atomic 95.xpm    ← kit definition (XML)
├── HipHop-Kit-GD Rawcutz Atomic 95.sxq    ← demo groove (optional)
├── HipHop-Kit-GD Rawcutz Block 92.xpm
├── HipHop-Kit-GD Rawcutz Block 92.sxq
│
├── HipHop-Kick-GD Atomic Kik.WAV          ← audio samples shared across kits
├── HipHop-Snare-GD Atomic Snr.WAV
├── HipHop-Hat-GD Atomic Hat.WAV
├── ...
│
├── Cache.json                              ← MPC browser cache (ignored by converter)
└── TagsCache.json                          ← MPC tag index (ignored by converter)
```

**`.xpm` files** — The kit definitions. Each one is an XML file that maps pad slots (A01 through H16) to sample names, sets mute groups, and lists any additional velocity layers. This is what the converter reads to know which WAVs go on which pads.

**`.wav` files** — The actual audio. They sit alongside the `.xpm` files in the same folder — not in sub-folders. The converter reads the WAV header (just the first 44 bytes) to get the sample length, then either copies the file or leaves it in place depending on your `--no-copy-wavs` setting.

**`.sxq` files** — Demo sequence files. Not every kit has one, but when present, the `.sxq` carries a pre-programmed groove for that kit — the same pattern that plays in the MPC's built-in demo. The converter reads the pad hits, velocities, and BPM from the `.sxq` and embeds them directly into the generated `.xpj` sequence so you can hear the kit in action as soon as you open it.

**`Cache.json` / `TagsCache.json`** — Internal MPC browser index files generated by the device software. The converter ignores them entirely.

> **Note:** Some expansions also contain a demo project file (e.g. `GoldDust Demo.xpj`) and a matching `GoldDust Demo_[ProjectData]/` folder with its own copies of certain WAVs. These are separate from the kit `.xpm` files and are not used by the converter.

## Tutorial

### Step 1 — Get the template file from your device

The converter needs a real `.xpj` file saved from your MPC device. This file provides the correct internal structure (mixer routing, instrument boilerplate) for your specific firmware version. Without it, generated projects won't load correctly.

**On your MPC device:**
1. Create a new empty project
2. Go to **Menu → Save Project As** and save it anywhere on your drive
3. Connect your MPC to your computer (USB storage or SD card)
4. Copy the `.xpj` file you just saved to the `template/` folder in this repo and rename it `device_template.xpj`

```
mpcsample-kit-converter/
└── template/
    └── device_template.xpj   ← your file goes here
```

> The `template/` folder is listed in `.gitignore` so your device file is never accidentally committed.

---

### Step 2 — Find your expansions

Run `--list-expansions` to see what's installed and what short names to use:

```bash
python3 mpc_kit_converter.py --list-expansions
```

Example output:

```
Expansion                                    Kits
───────────────────────────────────────────────────
golddust                                       28
nuskillz                                       20
soulprovider2                                  35
trapdrums                                       8
...
```

---

### Step 3 — Do a dry run first

Before writing anything to disk, use `--dry-run` to preview what will be generated. This is a good way to check BPM detection and confirm the kit count:

```bash
python3 mpc_kit_converter.py --expansion-name golddust --output ~/Desktop/kits --dry-run
```

Output will look like:

```
  DRY  HipHop-Kit-GD Rawcutz Atomic 95  [BPM=95  pads=16  demo=64hits]
  DRY  HipHop-Kit-GD Rawcutz Block 92   [BPM=92  pads=16  demo=58hits]
  ...
Done: 28/28 kits.
```

The `demo=Nhits` field means a `.sxq` demo groove was found for that kit and will be embedded in the sequence. Kits without one get a clean empty sequence.

---

### Step 4 — Convert

Run the same command without `--dry-run`:

```bash
python3 mpc_kit_converter.py --expansion-name golddust --output ~/Desktop/kits
```

---

### Step 5 — Copy to your MPC

Transfer the output folder to your MPC device via USB or SD card. In the MPC browser, navigate to the folder and you'll see one `.xpj` entry per kit — no sub-folders, no clutter.

Open any kit and hit **Play** — if a demo groove was found for that kit, it will play back immediately at the correct BPM.

---

### Converting everything at once

To convert all installed expansions in one go (this can take a while and copies a lot of audio):

```bash
python3 mpc_kit_converter.py --all-expansions --output ~/Desktop/kits --workers 4 --quiet
```

If your expansions are already on the MPC device and accessible at their original paths, you can skip copying WAVs to save disk space:

```bash
python3 mpc_kit_converter.py --all-expansions --output ~/Desktop/kits --no-copy-wavs
```

---

### Re-running after updates

By default the converter skips kits that already exist in the output folder. Use `--overwrite` to regenerate everything:

```bash
python3 mpc_kit_converter.py --expansion-name golddust --output ~/Desktop/kits --overwrite
```

## All Options

### Mode flags (pick one)

| Flag | Description |
|------|-------------|
| `--xpm FILE` | Convert a single `.xpm` kit file |
| `--expansion DIR` | Convert all kits in one expansion directory |
| `--expansion-name NAME` | Convert an expansion by its short name (e.g. `golddust`) — searches the Content directory automatically |
| `--all-expansions [DIR]` | Convert every expansion found in the Content directory |
| `--list-expansions` | Print a table of installed expansions and kit counts, then exit |

### Output & paths

| Flag | Default | Description |
|------|---------|-------------|
| `--output DIR` | *(required)* | Destination folder for generated kits |
| `--template FILE` | `template/device_template.xpj` | Path to the `.xpj` template file from your MPC device |
| `--content-dir DIR` | macOS: `/Library/Application Support/Akai/MPC/Content`<br>Windows: `C:\ProgramData\Akai\MPC\Content` | Override the MPC Content directory. Only needed if you've moved your content to a custom location. See [Understanding the source files](#understanding-the-source-files) for details. |

### Sequence & timing

| Flag | Default | Description |
|------|---------|-------------|
| `--bpm N` | *(auto)* | Override BPM for every kit. Normally BPM is read from the `.sxq` demo file if present, or auto-detected from the kit filename (e.g. `Kit 95 Am` → 95 BPM) |
| `--bpm-fallback N` | `120` | BPM to use when auto-detection finds nothing in the filename and no `.sxq` is present |
| `--bars N` | `2` | Sequence length in bars |

### Filtering

| Flag | Description |
|------|-------------|
| `--filter PATTERN` | Only convert kits whose name contains PATTERN (case-insensitive). Useful for testing a subset before a full run |
| `--skip PATTERN` | Skip kits whose name contains PATTERN (case-insensitive). Handy for excluding loop kits or specific categories |

### Behaviour

| Flag | Description |
|------|-------------|
| `--overwrite` | Replace existing `.xpj` files and WAVs in the output folder. Without this, already-converted kits are skipped |
| `--dry-run` | Preview what would be generated without writing anything to disk |
| `--no-copy-wavs` | Write `.xpj` files only — skip copying WAVs. Use this when the WAVs are already accessible on the device at their original Content directory paths |
| `--quiet` | Suppress per-kit output lines; show only the final summary counts |
| `--workers N` | Number of parallel workers for kit generation (default: `1`). Set to the number of CPU cores for fastest bulk conversion |
| `--version` | Print version number and exit |

## How it works

Akai expansion packs ship with `.xpm` files (XML) that define drum programs — which WAV file goes to which pad, mute group assignments, and layer configuration. This script:

1. Parses each `.xpm` to extract the 128-slot instrument map and mute groups
2. Reads each WAV file's header to get the exact frame count (so MPC knows where each sample ends)
3. Looks for a companion `.sxq` demo sequence file next to the `.xpm`. If found, the groove is translated into the project's sequence — pad hits, velocities, and BPM are all preserved. Notes that reference pads outside the kit's range are skipped cleanly.
4. Builds a `.xpj` project JSON using the device template as a structural base, injecting the pad assignments, sample data, and sequence
5. Gzip-compresses the result with the standard MPC `ACVS` header
6. Copies the kit's WAVs into a `_[ProjectData]` hidden folder alongside the `.xpj`
